> 00 / PIPELINE

HOW IT
WORKS.

Four stages. Each runs once per session, cached aggressively, and never touches the network after the TTL window.

01
DETECT

Profile the project

detect.sh walks the project: language, framework, package manager, 18 characteristics, and ~140 library signals across 10 languages.

→ project-profile.json
02
FEDERATE

Fetch the index

refresh_index.py hits every whitelisted source in parallel — marketplace.json, awesome-list walks, sitemap aggregators.

→ ~/.claude/skillforge/index.json
03
SCORE

Rank against profile

score.py multiplies profile × index. Tag matches, boosts, library anchors, default-for rules, and category clustering.

→ recommended / optional / hidden
04
AUDIT + INSTALL

Gate & materialize

Every candidate passes through the heuristic audit gate. Only clean skills land in ~/.claude/skills/.

→ selections.json (persistent)

> 02 / CAPABILITIES

SIX THINGS IT
DOES WELL.

01 · FEDERATED

No central catalog

Sources are whitelisted in config/sources.json. Marketplace.json, awesome-list READMEs, and sitemap aggregators all feed a single deduped index. Drop a source, add your own — full control.

02 · AUDITED

Gate before install

Every candidate passes the heuristic scanner. Markdown, shell, Python, JS, and filesystem rules. Optional Snyk adapter.

03 · OPT-OUT

Pre-checked UI

Recommended skills start ON. Skip what you don't want. Confirm installs the rest in one batch.

04 · DEEP INTROSPECTION

~140 libraries, 10 languages

Not just "it's a Python project" — SkillForge walks manifests and surfaces canonical library IDs. TanStack Query, Zustand, Drizzle, FastAPI, Pydantic, Flutter Bloc, Axum, Tokio, EF Core — all the way down. Skills can anchor to specific libraries.

05 · MULTI-AGENT

Export to Cursor & Codex CLI

Curate once, reuse everywhere. Cursor gets .cursor/rules/*.mdc files. Codex CLI gets a managed AGENTS.md section that preserves your user content.

06 · ZERO DEPENDENCIES

stdlib Python + bash + jq

No pip install. No node_modules. No database. Everything runs on what's already on a developer's machine. If you have Python 3.11 and jq, you're done.
